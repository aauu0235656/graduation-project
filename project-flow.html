图二 · RAG 问答完整流程
用户提问
👤 用户输入问题如："党员领导干部能否经商？"
🖥️ Vue.js 前端发送 POST 请求到 /api/rag/chat
HTTP POST / JSON Body
接口接收
RagController.chat()接收 { question, topK, sessionId }
参数校验@Validated + 非空检查
Spring Bean 调用 Service
业务处理
① 向量化查询
EmbeddingService.getEmbedding()调用阿里云百炼 text-embedding-v3 → 返回 1024 维向量
② 并行双路召回（同时执行）
🔍 路1：BM25 全文检索 SmartChineseAnalyzer → MultiFieldQueryParser → IndexSearcher.search() → Top-K 结果集 A
📐 路2：向量语义检索 遍历 HashMap 计算余弦相似度 cosθ = (a·b)/(|a||b|) → Top-K 结果集 B
③ RRF 融合排序
score = Σ 1/(k + rankᵢ)k=60，对两路结果去重合并后按分数降序排列，取最终 Top-K chunks
④ LLM 生成回答
拼接 Prompt【系统角色】+【检索上下文】+【用户问题】→ 调用 qwen-plus → 生成带引用的回答
返回 JSON 响应 + 异步保存历史
结果输出
📤 返回给前端{ answer, sources[], usedChunks[] }
💾 写入 qa_history 表异步保存本次问答记录供后续查询

图四 · 双路召回 + RRF 融合排序
查询输入
❓ 用户原始问题字符串如："党员领导干部能否经商办企业？"
EmbeddingService 向量化
🔍 BM25 全文检索通道
① SmartChineseAnalyzer对查询词进行中文分词
② MultiFieldQueryParser同时在 title 和 content 两个字段上构造布尔查询
③ IndexSearcher.search(query, topK)执行 Lucene 检索获取 Top-K 文档
📊 输出结果集 A[(docId₁, bm25Score), (docId₃, bm25Score), ...]
📐 向量语义检索通道
① 加载全部文档向量从内存 HashMap<id, float[1024]> 取出所有向量
② 遍历计算余弦相似度cosθ = (a·b) / (|a| × |b|)，O(n) 时间复杂度
③ PriorityQueue 取 Top-K按相似度降序排列
📊 输出结果集 B[(docId₂, vecScore), (docId₅, vecScore), ...]
两路结果汇入 RRF 排序器
RRF 融合
🔀 Reciprocal Rank Fusion 公式RRF(doc) = Σ 1/(k + rank_i(doc)) ，其中 k=60（平滑参数），i ∈ {BM25路, 向量路}
融合步骤
Step 1两路结果按 docId 合并，同一文档保留两条排名
Step 2对每个文档代入 RRF 公式求和得分
Step 3按 RRF Score 降序排序，取最终 Top-K
🎯 最终输出Top-K chunks 列表（含 content + sourceDocTitle + score）→ 送入 Prompt 拼接
