📋 关键技术要点说明

字段权重配置title 字段 Boost=3.0（标题匹配权重最高），category 字段 Boost=1.5（分类辅助筛选），content 字段为默认权重 1.0
中文分词器SmartChineseAnalyzer，基于词典的中英文混合分词，支持停用词过滤和同义词扩展
分类精确过滤categoryExact 使用 StringField 不分词存储，配合 BooleanQuery.MUST 实现精确类别约束
高亮策略Highlighter 对命中的 content 字段添加 <em> 标签，前端通过 CSS 样式渲染为红色粗体

model=text-embedding-v3
input=query Note right of API: 调用阿里云百炼 Embedding 接口
将自然语言转换为1024维稠密向量 API-->>ES: 返回 float[1024] 查询向量 qVec Note right of ES: 向量维度 d=1024
每个浮点数代表一个语义特征 Note over ES: === 步骤2：遍历计算余弦相似度 === ES->>VM: 遍历 vectorMap 所有条目 Note right of VM: key = chunkId (Long)
value = float[1024] 存储向量 loop 对每个 chunk 的存储向量 sVec ES->>ES: 计算余弦相似度 sim(qVec, sVec) Note right of ES: 公式：
sim = dot(q,s) / (|q| × |s|)
dot = Σ(q[i]×s[i])
|q| = √Σ(q[i]²) alt sim >= threshold (默认0.3) ES->>ES: 加入候选列表 else sim < threshold ES->>ES: 丢弃（语义不相关） end end Note over ES: 时间复杂度分析：
O(n×d)，n=chunk数量，d=1024维
一次遍历完成分子和模长计算 Note over ES: === 步骤3：排序截取 === ES->>ES: 候选列表按相似度降序排序 ES->>ES: 截取前 topK 个结果 ES-->>RS: 返回 List<Map> [{chunkId, similarity, score, ...}] Note over FS: === 数据持久化（后台） === ES->>FS: saveMapping() 序列化 HashMap → JSON 文件 Note right of FS: 写入磁盘 JSON 文件
解决内存数据重启丢失问题 Note over ES: === 启动加载 === FS->>ES: loadMapping() 反序列化 JSON → HashMap Note right of FS: 应用启动时从磁盘恢复
全部 chunk 向量数据

📋 关键技术要点说明

Embedding模型text-embedding-v3，输出 1024 维稠密向量，每个维度编码一个语义特征分量
相似度公式cos_sim(a,b) = Σ(a[i]·b[i]) / (√Σa[i]² × √Σb[i]²)，值域 [-1,1]，越接近1越语义相近
过滤阈值vector-threshold 默认 0.3，低于此值的 chunk 视为不相关直接丢弃，减少噪音干扰
向量存储方案HashMap<Long,float[]> 内存存储 + saveMapping()/loadMapping() JSON 持久化，兼顾性能和数据安全
时间复杂度暴力遍历 O(n×d)，n 为 chunk 数量，d=1024 为向量维度。当前数据量下响应在可接受范围
